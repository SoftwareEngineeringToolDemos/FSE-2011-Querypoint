Introduction to Debugging by lastChange

This page contains an object myObject. Whenever the user clicks on the page, myObject.myProperty value increases by 3 and the displayed value changes.

To see how lastChange works:

  1. Open Firebug (F12), then reload the page to update Firebug content
  2. Select Firebug Script panel
  3. set a breakpoint on line 17, where the value is written on the page.
  4. Reload the page and click on it. The breakpoint hits.
  5. In the Watch panel on the right side, check the value of this.myObject.myProperty. The value should be 3.
  6. Now right-click on myProperty and select lastChange. The panel will switch to QP
  7. In the QP panel you can see the location of the last change of myObject.myProperty and the program state at that point at the right side.
  8. You can ask for another, cascaded lastChange. From the TraceData panel on the right side select this.myObject.myProperty. Right click on myProperty and select lastChange. Now debugger shows a new location.
  9. All three locations, breakpoint, the first last change and the second last change are listed in the menu at the top of the panel.

Note: The implementation of lastChange is not yet complete for variables. We handle properties of variables but not the variable itself. In such cases you don't get any result from lastChange query.

myObject.myProperty = 0;
